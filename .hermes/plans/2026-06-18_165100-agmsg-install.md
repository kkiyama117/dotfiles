# agmsg Installation Plan — Claude Code + Hermes Bridge

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Install agmsg on Claude Code and set up the Hermes bridge so both agents can send/receive messages on shared agmsg teams.

**Architecture:** agmsg core (bash + sqlite3) gets installed as a Claude Code skill (`~/.agents/skills/agmsg/`). The Hermes side uses `agmsg-bridges` — a polling bridge that reads agmsg inbox and dispatches to Hermes, and vice versa. The bridge preloads an `agmsg-protocol` skill into Hermes.

**Tech Stack:** bash, sqlite3, Claude Code plugin system, Hermes skills, agmsg-bridges shell scripts

**Current State:**
- Claude Code v2.1.181: ✓ installed (`~/.local/bin/claude`)
- Hermes Agent v0.16.0: ✓ installed (`~/.local/bin/hermes`)
- sqlite3 3.42.0: ✓ on PATH (`/opt/android-sdk/platform-tools/sqlite3`)
- bash 5.3.15: ✓
- agmsg: ✗ not installed
- agmsg-bridges: ✗ not cloned

### Task 4: Adapt Bridge for Manjaro Linux

**Objective:** The agmsg-bridges scripts assume macOS paths. Fix paths for Manjaro Linux.

**Files likely to change:**
- Modify: `~/projects/agmsg-bridges/bin/agmsg-bridge.sh` — fix `agmsg` binary path
- Modify: `~/projects/agmsg-bridges/bin/setup.sh` — fix paths

**Step 1: Identify macOS-specific paths**

```bash
grep -rn '/opt/homebrew' ~/projects/agmsg-bridges/bin/
grep -rn 'brew' ~/projects/agmsg-bridges/bin/
```

**Step 2: Replace with Linux-appropriate paths**

On Manjaro, agmsg is invoked via the scripts directly:
```bash
# Instead of /opt/homebrew/bin/agmsg, use:
AGMSG_DIR="$HOME/.agents/skills/agmsg"
agmsg() { "$AGMSG_DIR/scripts/send.sh" "$@"; }
```

Or better: create a wrapper script at `~/.local/bin/agmsg`:
```bash
#!/bin/bash
# agmsg wrapper — delegates to installed scripts
SKILL_DIR="$HOME/.agents/skills/agmsg"
case "${1:-}" in
  send)    shift; exec "$SKILL_DIR/scripts/send.sh" "$@" ;;
  inbox)   shift; exec "$SKILL_DIR/scripts/inbox.sh" "$@" ;;
  history) shift; exec "$SKILL_DIR/scripts/history.sh" "$@" ;;
  team)    shift; exec "$SKILL_DIR/scripts/team.sh" "$@" ;;
  *)       echo "Usage: agmsg {send|inbox|history|team} ..." >&2; exit 1 ;;
esac
```

**Step 3: Test the wrapper**

```bash
~/.local/bin/agmsg team test-team
```
Expected: lists team members.

## Risks & Tradeoffs

1. **macOS-centric paths in agmsg-bridges**: The bridge scripts assume `/opt/homebrew/bin/agmsg`. On Linux we need a wrapper or path fixes. Task 4 handles this.

2. **sqlite3 from Android SDK**: The current sqlite3 is at `/opt/android-sdk/platform-tools/sqlite3`. This works but is unusual. Consider installing system sqlite3: `sudo pacman -S sqlite` if issues arise.

3. **Polling latency**: The bridge polls at intervals (15-30s default). Real-time push only works Claude→Claude (monitor mode). Hermes will have polling latency.

4. **Bridge is community-maintained**: agmsg-bridges is not part of core agmsg. It may lag behind agmsg updates.

5. **No native Hermes integration yet**: Hermes doesn't have a built-in `/agmsg` slash command. The bridge + skill approach is a workaround.
